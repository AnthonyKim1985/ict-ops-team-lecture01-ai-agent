Part 01

AI 에이전트란?

챗봇과는 무엇이 다를까요?

❝

“내일 항공권 예약해줘”

챗봇은 말은 잘하지만,
스스로 사이트에 들어가 예약·결제까지 하진 못합니다.

🎯 한 문장 정의

AI 에이전트는 목표를 받으면 스스로 계획하고, 도구를 사용해, 행동하는 AI입니다.

챗봇 = 대답하는 AI  ·  에이전트 = 일을 해내는 AI

🤝 쉬운 비유

챗봇 vs 에이전트, 사람으로 비유하면?

💬

챗봇 = 아는 게 많은 친구

물어보면 척척 대답해 줍니다.
하지만 “대신 처리해 줘”는 못 해요.

🧑‍💼

에이전트 = 똑똑한 인턴

일을 맡기면 알아서 검색하고,
정리해서 결과까지 가져옵니다.

⚖️ 챗봇 vs 에이전트

챗봇 대화형

  • 질문 1번 → 답변 1번
  • 외부 도구를 사용하지 못함
  • 매번 사람이 다음 단계를 지시
  • 한 번의 대화로 끝나는 단발성

에이전트 실행형

  • 목표 1번 → 여러 단계 스스로 수행
  • 검색·계산 등 도구로 직접 실행
  • 다음 행동을 스스로 판단
  • 목표 달성까지 끝까지 진행

🧠 부품 ① 두뇌 — LLM

Large Language Model (거대 언어 모델)

  • ChatGPT, Claude 같은 모델이 바로 LLM입니다
  • 상황을 보고 “지금 무엇을 할까”를 판단
  • 사람의 두뇌처럼 다음 행동을 결정하는 중심
  • 단, 혼자서는 생각만 — 행동하려면 도구가 필요

🔨 부품 ② 도구 — Tools

LLM의 손과 발 — 실제로 무언가를 해내는 장치

  • 웹 검색 — 최신 정보를 찾아온다
  • 계산기 — 정확한 계산을 한다
  • 이메일·캘린더 — 보내고 일정을 확인한다
  • 외부 프로그램(API) — 다른 서비스를 부른다
  • 도구가 많을수록 할 수 있는 일이 늘어난다

💾 부품 ③ 메모리 — Memory

겪은 일을 기억해 맥락을 유지

  • 단기 기억 — 지금 이 대화에서 방금 한 일
  • 장기 기억 — 이전 작업, 사용자의 취향과 기록
  • 덕분에 “아까 찾은 결과”를 다음 행동에 활용
  • 맥락을 잃지 않고 여러 단계를 이어간다

🗺️ 부품 ④ 계획 — Planning

큰 목표를 작은 단계로 쪼개기

예) “여행 준비해 줘” 라는 큰 목표가 들어오면…

  • ✅ 항공권 알아보기
  • ✅ 숙소 찾기
  • ✅ 일정 짜기
  • 사람이 “할 일 목록”을 만드는 것과 똑같다

🧶 ReAct 쉽게 이해하기

Reasoning(추론) + Acting(행동) = ReAct

생각만 하지도, 무작정 행동만 하지도 않습니다. 생각 → 행동 → 결과 확인 → 다시 생각. 번갈아 하기 때문에 중간에 틀려도 스스로 고쳐가며 목표에 다가갑니다.

Part 04

실제 사례

이미 우리 가까이 와 있습니다

💻 사례 ① 코딩 에이전트

Claude Code, Cursor … (이 슬라이드도 이렇게 만들어졌어요!)

  • “로그인 기능 만들어 줘”라고 말로 요청
  • 에이전트가 코드를 작성하고 파일을 수정
  • 직접 실행해 보고 오류까지 스스로 수정
  • 비전공자도 설명만으로 간단한 프로그램 제작

🔎 사례 ② 리서치·검색 에이전트

자료 조사와 요약을 대신해 줍니다

  • “OO 시장 동향 조사해 줘” 한마디면 시작
  • 여러 웹사이트를 돌며 자료를 수집
  • 중요한 내용만 골라 요약 보고서 작성
  • 몇 시간 걸릴 조사를 몇 분으로 단축

📧 사례 ③ 업무 자동화 에이전트

반복적이고 귀찮은 일을 대신 — 디지털 비서

  • 쌓인 이메일을 중요도별로 분류
  • 회의 일정을 잡고 캘린더에 등록
  • 매주 반복되는 보고서를 자동 작성
  • 사람은 더 창의적인 일에 집중

🤝 사례 ④ 여러 에이전트의 협업

Part 05

직접 보기

아주 간단한 데모 — 겁먹지 마세요!

💻 에이전트는 이런 모습

외울 필요 없어요 — ‘반복 고리’ 구조만 봐주세요

simple_agent.py 
# 아주 단순하게 표현한 AI 에이전트
목표 = "내일 서울 날씨 알려줘"
기억 = []
끝남 = False

while not 끝남:
    생각 = LLM.판단(목표, 기억)        # 1) 무엇을 할지 생각
    if 생각.도구가_필요하면:
        결과 = 도구_실행(생각.도구)     # 2) 도구 사용 (검색·계산 등)
        기억.append(결과)             # 3) 결과를 관찰하고 기억
    else:
        답변 = 생각.최종_답변          # 4) 충분하면 답을 만든다
        끝남 = True

print(답변)

📝 날씨 에이전트 만들기 — 프롬프트

코딩 에이전트(예: Claude Code)에게 이렇게 부탁하면 됩니다

🤖 weather_agent_prompt

목표 “날씨 에이전트”를 만들어 줘. 사용자가 도시 이름을 말하면 그 도시의 내일 날씨를 알려주는 프로그램이야.

도구 인터넷 날씨 검색 API를 사용해서 (도시 이름 → 기온·강수 정보)를 가져와.

행동 ① 질문에서 도시 이름 찾기 → ② 날씨 도구 호출 → ③ 결과를 한 문장으로 정리 → ④ 우산이 필요하면 한마디 덧붙이기

출력 예시 “내일 서울은 12°C에 비가 와요. 우산 챙기세요! ☂️”

색깔 라벨이 곧 오늘 배운 부품 — 목표·도구·행동·출력 그대로죠!

⚠️ 한계와 주의점

  • 환각 — 그럴듯하지만 틀린 정보를 말할 수 있다
  • 비용·속도 — 여러 단계를 거쳐 느리고 비용이 든다
  • 신뢰성 — 매번 똑같이 잘하지는 않는다
  • 보안·안전 — 실제 행동을 하므로 실수가 피해로

🧑 최종 감독은 사람의 몫입니다.

⭐ 핵심 3가지 요약

  • 스스로 행동하는 AI — 대답이 아니라 일을 ‘해낸다’
  • 네 가지 부품 — 두뇌(LLM) · 도구 · 메모리 · 계획
  • 생각 → 행동 → 관찰을 반복하며 목표에 다가간다
